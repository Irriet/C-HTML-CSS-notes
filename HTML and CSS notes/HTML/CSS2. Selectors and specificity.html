Selectors and specificity

HTML is a structure, while CSS is styling. They're disconnected, so a way to assign the style to proper elements is needed.

Selectors are the way to grab and manipulate HTML. There are many different ways to select, however they all provide the same outcome. Different selectors have different uses.

Basic selectors

The element selector (inline)

It targets a single element.

(in HTML:)

<h2 style="color: pink;">A good title</h2>

The class selector

Targets elements with a certain class name.

(in CSS:)

.someclass {color: blue;}

(in HTML:)

<h2 class="someclass">A subheading.</h2>

The ID selector

It's about the same as class selector, but by convention it's used on only one element at a time. It will work for more, but generally it's not used this way.

(in CSS:)

#sometext {color: orange;}

(in HTML:)

<p id="sometext">Some text.</p>

Specificity and when to use selectors

Different selectors are more powerful than others. Element selectors are less powerful than class ones, so they get overridden. The general rule is:

element < class < id < inline styling


Pseudoselectors

They're commonly used in interactivity with the website - they're only visible when something is done on the website. A good example is hovering.

Hover

h2:hover {
color: purple;
}

Visited and not visited link

#google-link:link {
color: blue;
}

#google-link:visited {
color: red;
}

First child

Here's an explanation of which child is which:

<tr>
 <td>A column</td> - first child
 <td>Another column</td> - second child
 <td>Unbelievably, a third one</td> - last child
</tr>

Below an example of some styling with first child:

li:first child {
color: green;
}

<ul>
 <li>This is the list.</li>
 <li>This is the list.</li>
  <li>This is the list.</li>
 <li>This is the list.</li>
 <li>This is the list.</li>
 <ol>
  <li>It even gets numbered.</li>
  <li>It even gets numbered.</li>
 </ol>
</ul>

Last child

It does the same as first child, but for the last element in the container.

N-th child

Affects n-th child.

li:nth-child(2) {
color: green;
}

<ul>
 <li>This is the list.</li>
 <li>This is the list.</li>
  <li>This is the list.</li>
 <li>This is the list.</li>
 <li>This is the list.</li>
 <ol>
  <li>It even gets numbered.</li>
  <li>It even gets numbered.</li>
 </ol>
</ul>

Only child

li:only-child {
color: green;
}

Note that it won't work if there is more than 1 child.


Advanced selectors

These can contain few selectors at once.

Adjacent sibling selector

It targets every single element that directly follows a different element. These will be styled even if they aren't in the same parent element.

To colour every anchor tag that follows a h2:

h2 + a {
color: cornflower blue;
}
For:
<h2 id="services">Services.</h2>
<a href="https://www.google.com">Imagine clicking this to go to Google.</a>

General sibling selector

Siblings have to be sharing the same parent. They also have to directly follow each other.

Textarea ~ button {
color: red;
}
For:
<form>
[...]
<textarea></textarea>
<button>Submit</button>
</form>

Child selector

For every single child of some element. For example, for every single li inside of an ul:

ul > li {
color: red
}

Note that li:only-child would be more specific than the ul > li


Attribute selector

It's used to select elements that have certain values of attributes. The general rule is: h2[attribute=value]. So src after img is also an attribute. For example:

h2[class=subtitle]{
color: blue;
}

With the code above, every h2 that has a class of subtitle will be affected.

Selecting an image with certain source:

img[src=”../img/”] {
border: 10px solid black;
}

Instead of simple = sign, you can use variations to get different results.

  • ^= for the path beginning with a certain element
  • $= for a path ending with a certain element
  • *= for a path containing mentioned element anywhere
